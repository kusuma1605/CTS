# Module 10 - Version Control - GIT

## What this demonstrates
- `git-exercises/01-basic-commands.md` - `git init`, staging, committing,
  `git diff` vs `git diff --staged`, and reading `git log`
- `git-exercises/02-branching-and-merging.md` - creating branches,
  producing and resolving a real merge conflict, `--no-ff` merges, and a
  simplified Git Flow (`main` / `develop` / `feature/*`)
- `git-exercises/03-remote-and-collaboration.md` - connecting a local repo
  to GitHub, forking a repository, the `origin`/`upstream` remote
  convention, and opening a Pull Request
- `.gitignore.sample` - a starting `.gitignore` covering the .NET, Angular,
  IDE, and OS artifacts you'll want to keep out of every repo you create
  across this program

## To run
1. Work through the three exercise files in order in a scratch folder -
   each one builds on the repo state left by the previous one.
2. Answer the "Questions to answer" at the end of each exercise in your
   own words before moving on; they're designed to check understanding,
   not just command recall.
3. Use `.gitignore.sample` (renamed to `.gitignore`) in the solution
   repository you create per the handbook's **Submitting Solutions**
   instructions.
