# Scrum Quick Reference

## Roles
| Role | Responsible for |
|---|---|
| Product Owner | Owns and prioritizes the Product Backlog; represents stakeholder/customer value |
| Scrum Master | Facilitates ceremonies, removes blockers, protects the team's process |
| Development Team | Delivers the Increment; self-organizes on how to build it |

## Ceremonies
| Ceremony | Purpose | Typical length (2-week sprint) |
|---|---|---|
| Sprint Planning | Select and commit to backlog items for the sprint | ~2-4 hours |
| Daily Scrum | Sync on progress and blockers | 15 minutes |
| Sprint Review | Demo the Increment to stakeholders, gather feedback | ~1-2 hours |
| Sprint Retrospective | Inspect and adapt the team's own process | ~1 hour |

## Artifacts
| Artifact | What it is |
|---|---|
| Product Backlog | The full, ordered list of everything that might be built |
| Sprint Backlog | The subset of the Product Backlog selected for the current sprint, plus a plan to deliver it |
| Increment | The sum of all completed backlog items, meeting the Definition of Done |
| Definition of Done | The team's agreed checklist for "actually finished" (e.g. coded, tested, reviewed, merged) |

## Self-check exercise
Write your own team's Definition of Done for a backend API story (like
"Add to cart" from the backlog) - list at least 5 concrete, verifiable
conditions (e.g. "unit tests pass with >80% coverage", "peer-reviewed and
approved on the PR", "Swagger docs updated").
