# Module 13 - Agile Methodology

## What this demonstrates
- `agile-artifacts/product-backlog.md` - a realistic 6-item backlog for a
  sample "DeepSkilling Store" project built on the Products/Orders domain
  from Modules 5-8, with one story fully worked through the INVEST
  principle and Given-When-Then acceptance criteria, plus two stories left
  for you to write yourself
- `agile-artifacts/sprint-planning-exercise.md` - a solo Planning Poker
  simulation, a velocity-constrained sprint-planning exercise, a burndown
  sketching exercise (ideal vs. actual), and retrospective-style
  reflection questions
- `agile-artifacts/scrum-quick-reference.md` - a one-page reference table
  of Scrum roles, ceremonies, and artifacts, plus a self-check exercise to
  write your own Definition of Done

## To run
1. Read `scrum-quick-reference.md` first if Scrum roles/ceremonies/artifacts
   are still new terms.
2. Complete `product-backlog.md`'s "Your turn" section (stories 3 and 4)
   before moving on - writing your own is more valuable than only reading
   the worked example.
3. Work through all four steps of `sprint-planning-exercise.md` in order.
4. There's no code to run in this module - treat the markdown files as
   worksheets, and compare your answers with a peer or your SME Connect
   session if possible.
